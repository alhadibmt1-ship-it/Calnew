Project: calcsmart24.com

Goal:
Upgrade the calculator website so it can scale to 1000+ SEO-optimized calculator pages using a dynamic template system instead of manually coded pages.

Required Improvements:

1. Dynamic Calculator Template
   Create a universal calculator page template that loads calculator data dynamically.
   The template should include:

- Page title (SEO optimized)
- Calculator input fields
- Calculation formula logic
- Result output
- Explanation section
- Example calculation
- FAQ section
- Related calculators
- Internal links

The template must support hundreds or thousands of calculators.

2. Calculator Database
   Move all calculators into a structured JSON or database file.

Example structure:

{
"name": "BMI Calculator",
"slug": "bmi-calculator",
"category": "health",
"inputs": ["weight", "height"],
"formula": "weight / (height * height)",
"description": "Calculate Body Mass Index",
"example": "Example: weight 70kg and height 1.75m",
"related": [
"ideal-weight-calculator",
"tdee-calculator",
"body-fat-calculator"
]
}

The system should dynamically generate pages using the slug.

Example URL structure:

/construction/concrete-slab-calculator
/finance/loan-calculator
/health/bmi-calculator
/convert/meters-to-feet

3. Category Structure
   Create category pages for SEO:

/construction/
/finance/
/health/
/science/
/unit-converters/
/lifestyle/
/business/
/education/
/time/

Each category page should list all calculators in that category.

4. Automatic Internal Linking
   Each calculator page should automatically display:

- 5 related calculators
- 5 popular calculators
- 5 calculators from the same category

This improves crawling and SEO.

5. SEO Optimization
   Each page must include:

- Unique meta title
- Meta description
- H1 title
- Structured headings
- Schema markup (FAQ + WebApplication schema)
- Clean URL slugs
- Fast loading design

6. Automatic Sitemap Generation
   Create an automatic sitemap generator.

When new calculators are added to the database, they should automatically appear in:

/sitemap.xml

The sitemap should support more than 5000 URLs.

7. Unit Converter Expansion
   Add a system for unit-pair pages.

Examples:
meters-to-feet
feet-to-meters
kg-to-lbs
lbs-to-kg
celsius-to-fahrenheit
fahrenheit-to-celsius

These should be generated automatically using the same template.

8. Performance Optimization
   Optimize the website for speed:

- Lazy loading
- Minimal JavaScript
- Cloudflare cache friendly
- Mobile responsive UI

9. Calculator Count Expansion
   Generate placeholder entries in the database for at least 1000 calculators including:

- Construction calculators
- Finance calculators
- Health calculators
- Unit converters
- Business calculators
- Lifestyle tools

10. Navigation Improvements
    Add:

- Search bar for calculators
- Category filters
- Popular calculators section
- Recently added calculators section

Goal Result:
The website should support unlimited calculator pages and easily scale beyond 1000 tools while remaining SEO optimized and fast.

The system should automatically generate pages from the database instead of manually coding each calculator page.
